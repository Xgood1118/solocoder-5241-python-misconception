学校老师改完卷只能打个分，错题原因全凭印象回忆，特别不靠谱。每次考完试想看看到底是学生审题不仔细、概念没搞懂、还是真不会，靠脑子记根本记不过来。Python 用 FastAPI 写一个本地能起的小工具，每次考试错题能录进去做点像样的分析。

录错题这块要做得顺手。老师改卷时看到一道错题，界面上点几下就能录：哪个学生错的（学号、姓名都得要，重名太多了）、题目原文（粘贴长文本就行，不做 OCR）、错因归到几个大类——计算错误、概念不清、审题错误、不会做这四种，每种下面能再写一句具体说明，比如概念不清可以写「混淆了加速度和速度」、审题错误可以写「没看到单位换算」。每道错题挂上对应知识点标签，一道题可能挂好几个知识点，都能录。

录完后能做的事挺多。按错因统计每次考完试一定要有，全班多少栽在审题、多少是真不会，哪道题错的人最多，错的人都错在哪，这种统计按班级、知识点、错因三个维度都能切。按学生看个人错题本，每个学生历史错题按知识点聚合一下，哪些知识点反复错一眼看到。

班级共性错题和个人错题要分开。共性错题是全班一半以上都错的，说明是教学问题；个人错题是某几个学生反复栽在同一类知识点上，说明是个人薄弱环节。两种情况分开列，老师看到共性错题能回去调整讲课重点，看到个人错题能针对性辅导。

高频错题识别要有一手。一道题被不同学生在不同考试里反复错，说明题目本身或者对应知识点讲解有问题，跟纯偶然错不一样。统计时同一道题用题目的文字内容做个简单指纹去重，别因为录入时多打了一个空格就当成两道题。

导出功能要有。分析完老师想把报告带走，导成 HTML 文件就行，文件里包含班级整体情况、各知识点错题分布、共性错题清单、个人错题本这几块，打开浏览器直接看。

涉及几个模块要分清楚：错题录入模块管题目原文和错因的关联，知识点模块管标签体系（一个知识点下面挂哪些题目），学生模块管基本信息，分析模块负责按维度聚合统计，导出模块把结果拼成 HTML。关联上题目和学生是多对多（一道题多个学生错，一个学生多道题错），错题和知识点也是多对多。多步流程体现在：录错题 → 按错因分类 → 按班级/知识点聚合 → 识别共性/个性错题 → 输出报告。

录入时如果一道题没挂知识点，要提示老师补一下别让分析的时候少数据；学期中途学生转班了，错题数据要跟着学生走不能跟着班级走。报告里共性错题的判定阈值要能让老师调（默认是 50% 但不同班级规模不一样）。

运行环境用 FastAPI 起本地服务，前端简单 HTML 页面就行，数据存本地 JSON 文件就行。
